# Sprint Research

Run Phase 1 of Sprint Protocol.

1. Read `../sprint-protocol/SKILL.md`.
2. Read `../sprint-protocol/references/codex-orchestration.md`.
3. Read `../sprint-protocol/references/phase-1-research.md`.
4. Use `update_plan`.
5. Classify the intake as raw or baked.
6. Review gaps, inconsistencies, missing decisions, and untestable criteria.
7. Decide whether the work fits one sprint.
8. If it fits, create `research.md` and `plan.md` in one sprint folder.
9. If it does not fit, present a multi-sprint distribution, ask for approval, then create independent sprint folders with their own `research.md` and `plan.md`.
10. Spawn `explorer` agents for independent research or planning-review concerns.

Treat legacy `/sprint-research <name>` as this skill.
